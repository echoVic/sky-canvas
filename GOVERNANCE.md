# 项目治理

本文档描述 Sky Canvas 项目的治理结构与决策流程。

## 角色

### 维护者（Maintainers）

维护者对项目拥有完整的写入权限，负责：
- 审阅和合并 Pull Request
- 发布新版本
- 管理 Issue 和 Discussions
- 制定技术方向

当前维护者名单见 [MAINTAINERS.md](./MAINTAINERS.md)。

### 贡献者（Contributors）

任何提交被合并的 PR 的人都是贡献者。贡献者可以：
- 提交 Issue 和 PR
- 参与 Discussions
- 审阅其他 PR（非正式）

### 用户（Users）

使用项目的任何人。用户可以：
- 提交 Issue 和功能建议
- 参与 Discussions

## 决策流程

### 日常决策

- 单个维护者可以合并符合贡献指南的 PR
- 如有争议，需要至少两位维护者同意

### 重大决策

以下决策需要维护者多数同意：
- 架构变更
- 破坏性 API 变更
- 新增/移除核心依赖
- 发布策略变更

### 争议解决

1. 在相关 Issue/PR 中讨论
2. 如无法达成共识，由维护者投票决定
3. 票数相同时，由项目负责人决定

## 成为维护者

贡献者可以通过以下方式成为维护者：
1. 持续贡献高质量代码（至少 3 个月）
2. 积极参与代码审阅
3. 获得现有维护者提名
4. 获得维护者多数同意

## 行为准则

所有参与者必须遵守 [行为准则](./CODE_OF_CONDUCT.md)。
